---
title: "3.3 由等价类构造整数"
description: "用自然数对的等价类去构造整数，并把每个等价类读成一个有符号数。"
---

讲义把整数建成一个关系下的等价类。想法很简单：与其硬造一个全
新的 `-1`，不如让像 `(0, 1)` 或 `(3, 4)` 这类对子去代表同一个有
符号数。

## 那条关系

<Definition title="以等价类定义整数">
在 `N^2` 里工作，也就是自然数有序对的集合。

定义关系 `~_Z` 为

`(a, b) ~_Z (c, d)` 当且仅当 `a + d = b + c`。

整数就是这条关系的等价类。
</Definition>

你可以把 `(a, b)` 读成差值 `a - b`。所以即使两个对子不同，只要
它们表达的是同一个差值，它们就代表同一个整数。

## 具体例子

<WorkedExample title="两个对子可以代表同一个整数">
`(3, 1)` 和 `(5, 3)` 是等价的，因为

`3 + 3 = 1 + 5`。

所以它们代表同一个整数。在一般直觉里，两者都代表 `2`。
</WorkedExample>

<CommonMistake title="不要只看到对子本身">
整数不是那个有序对本身。那个对子只是等价类的一个代表元。
</CommonMistake>

## 快问快答

<QuickCheck prompt="在 `~_Z` 下，`(2, 0)` 和 `(5, 3)` 是否等价？">
用 `a + d = b + c` 这条规则来检查。
</QuickCheck>

<RevealSolution title="答案">
是。`2 + 3 = 0 + 5`，所以这两个对子等价。
</RevealSolution>

<QuickCheck prompt="为什么要用等价类，而不是直接用原本的对子？">
用一句话回答。
</QuickCheck>

<RevealSolution title="答案">
因为很多不同对子都应该代表同一个有符号数，而等价类正好把这些代表元收在一起。
</RevealSolution>
