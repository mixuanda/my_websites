整数是由自然数对构造出来的。有理数也用同样想法，只是第二个分
量不能是 `0`。

## 商模型

<Definition title="有理数">
设

`Y = Z × (Z \ {0}) = {(a, b) | a in Z, b != 0}`。

定义 `(a, b) ~_Q (c, d)`，当且仅当 `a d = b c`。

有理数就是等价类 `Y / ~_Q`。
</Definition>

你可以把 `[(a, b)]` 当成分数 `a/b`。做商集合的目的，就是让相等
的分数在定义上就相等。

## 为什么要顾及良定性

如果你只在代表元上写公式，就要先确认答案对同一个有理数的每个
代表元都一样。这就是这里的“良定”意思。

<WorkedExample title="加法仍然符合分数直觉">
讲义把加法定义为

`[(a, b)] + [(c, d)] := [(ad + bc, bd)]`。

例如，

`[(1, 2)] + [(1, 3)] = [(5, 6)]`。

如果把第一个代表元换成 `(2, 4)`，公式会得到 `[(10, 12)]`，
而这和 `[(5, 6)]` 是同一个有理数。
</WorkedExample>

<CollapsibleProof title="为什么加法公式是良定的">
假设 `(a, b) ~_Q (a', b')` 和 `(c, d) ~_Q (c', d')`。
这表示 `a b' = b a'` 和 `c d' = d c'`。

讲义的证明会把这两个等式整理起来，推出

`(ad + bc, bd) ~_Q (a'd' + b'c', b'd')`。

所以加法不会依赖你选哪一组代表元。
</CollapsibleProof>

<CommonMistake title="不要用忽略符号的方式去比分数">
Worksheet 5 会问像 `p/q R m/n if pn - qm > 0` 这种关系。对任意代表元
来说，这类比较法不安全，因为换一个等价代表元时，符号可能改变。
</CommonMistake>

## 快问快答

<QuickCheck prompt="Worksheet 5 里哪一条关系在 Q 上是良定的：`p/q R m/n` if `p > m`，if `pn - qm > 0`，还是 if `(pn - mq)nq > 0`？">
想想把代表元换成等价形式会发生什么事。
</QuickCheck>

<RevealSolution title="答案">
按题目写法，只有第三条是良定的。前两条都会依赖你选的代表元。
</RevealSolution>

<QuickCheck prompt="为什么 `Y` 要排除分母是 `0` 的对子？">
用数学理由回答，不要只说直觉。
</QuickCheck>

<RevealSolution title="答案">
因为第二个分量要扮演分母，而这个构造不能把除以 `0` 放进去。
</RevealSolution>
