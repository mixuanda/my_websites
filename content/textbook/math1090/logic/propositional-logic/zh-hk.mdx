# 命題邏輯

先從能判斷真或假的陳述開始。這就是本課最基本、也最實用的邏輯單位。

## 甚麼是命題？

<Definition title="命題">
命題是有確定真假值的陳述。它不是真，就是假。

例子：

- `2 + 2 = 4` 是命題。
- `請關門！` 不是命題，因為它是命令。
- 若未指定 `x`，`x + 1 = 3` 也還不是命題。
</Definition>

<CommonMistake title="含有變量的句子不會自動變成命題">
如果陳述仍然依賴某個未指定的變量，它的真假值就未固定。要先知道變量代表甚麼，才可以判斷真假。
</CommonMistake>

## 基本連接詞

課程會反覆用到五個連接詞：

| 記號 | 讀法 | 核心意思 |
| --- | --- | --- |
| `¬P` | 非 `P` | 將真假值取反 |
| `P ∧ Q` | `P` 且 `Q` | 只有兩邊都真才真 |
| `P ∨ Q` | `P` 或 `Q` | 只要至少一邊真就真 |
| `P → Q` | 若 `P`，則 `Q` | 只有 `P` 真而 `Q` 假時為假 |
| `P ↔ Q` | `P` 當且僅當 `Q` | 兩邊真假一致才真 |

## 把公式讀成句子

<WorkedExample title="把蘊含讀出來">
設 `P` 代表「正在下雨」，`Q` 代表「地面是濕的」。

`P → Q` 讀作：

「若正在下雨，則地面是濕的。」

這句話**不**表示現在一定正在下雨，只是說一旦前件成立，後件就必須成立。

<RevealSolution title="初學時最容易誤讀的地方">
`P → Q` 只會在一種情況下為假：`P` 真而 `Q` 假。其餘情況都是真。
</RevealSolution>
</WorkedExample>

## 先做一個小檢查

<QuickCheck prompt="下列哪些是命題？`2 + 2 = 4`、`請打開窗戶。`、`x + 1 = 3`">
`2 + 2 = 4` 是命題。`請打開窗戶。` 不是。`x + 1 = 3` 要先指定 `x` 才算命題。
</QuickCheck>
