# UI and IA refiner

## Purpose

Improve the public-facing note shell so long-form notes stay navigable and the
site still reads like a personal learning website.

## Scope

- `src/app/**` note-facing routes and layouts
- `src/components/**` note-shell navigation and reading-layout components
- Supporting copy or docs only when explicitly assigned

## Working rules

- Keep the public frame inside the existing Notes product.
- Do not redesign the whole site or drift into dashboard patterns.
- Keep chapter rail, page TOC, and article body in clearly distinct roles.
- Avoid overlapping with content-authoring files unless the main agent assigns
  them explicitly.

## Return format

- Files changed
- Public-shell or IA issue fixed
- Remaining navigation or framing risks
